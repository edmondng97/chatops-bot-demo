核心理念

能用确定性死流程做掉的，门卫（无 LLM）全包；只有真正需要分析的部分才交给 AI worker。

确定性优先

解析 / 路由 / 向导 / 生命周期全在确定性门卫里，不会退化。

一线程一实例

一个 thread = 一个命令实例，从生到死，线程隔离是铁律。

config 驱动可扩展

新增能力 = 加一份 JSON + 一个 worker skill，不碰路由 / 生命周期。

一条命令从入口到回复。点击任意一步高亮查看职责。

代码规范 & 技术决策

工程纲领与关键权衡 —— 面试讲解锚点。

工程纲领

  • kebab 文件名 · Pascal 类 · camel 方法
  • interfaces/{module} 桶导出，禁深路径 import
  • 禁 forwardRef · 禁跨边界 Service→Service
  • i18n 每个 key en/zh 并排，漏一种语言编译失败
  • 注释用英文 · 对用户解释用简体中文

技术权衡

  • 持久层 vs 缓存队列分工：业务真相落数据库（按 threadKey upsert + 唯一索引）；Redis 只配做 cache + queue，绝不当可靠存储。
  • threadKey 不用 thread_id：thread_id 只在话题回复上出现、根消息事件里没有，会把一个话题劈成两个 key；用 root_id ?? message_id 才稳定。
  • fail-fast 不降级：持久层不可用则门卫启动失败（进程管理器重启兜底），运行期绝不退回文件存储。
  • worker 纯大脑：不持凭证、不碰 IM、崩溃只死自己；报告统一由 broker 出站。
